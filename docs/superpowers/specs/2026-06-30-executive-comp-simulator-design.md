# 役員報酬・会社員比較シミュレーター 設計書

作成日: 2026-06-30

## 1. 目的

Web 上で以下 3 ケースを比較できるシミュレーターを作る。

1. 普通の会社員として働いた場合
2. 法人化して役員報酬を受け取る場合
3. 法人化し、役員報酬・事前確定届出給与・社宅・iDeCo+・小規模企業共済・出張旅費などを使った場合

表示指標: 個人手取り / 実質手取り / 法人残キャッシュ / 個人＋法人合計 / 所得税 / 住民税 / 社会保険料(本人・会社) / 法人税等 / 会社員との差額 / 各節税策の効果。

## 2. 技術構成

- Next.js (App Router) + TypeScript
- Tailwind CSS
- React Hook Form + Zod
- Recharts
- decimal.js（金額計算は全て Decimal。表示時のみ丸める）

`src/lib/calc/*` は UI 非依存の純粋関数として実装し、ユニットテスト対象とする。

## 3. 確定した設計判断（ブレインストーミング結果）

1. **標準報酬月額表は料率から自動生成する。** 全 50 等級の表をハードコードせず、等級区分（min/max/standardMonthly の固定値）に料率を掛けて健保・介護・厚年の本人/会社負担を実行時に算出する。料率を 1 か所変えれば全等級が再計算され、年度・都道府県の差し替えが容易。
2. **MVP を一気に実装する。** 会社員 vs 法人役員 + iDeCo+ + 小規模企業共済 + 社宅概算 + 出張旅費 + 最適化モードまで、計算ロジック＋UI を通しで実装する。
3. **計算は「概算」と割り切り、ユニットテストで回帰を防ぐ。** 公式値との厳密突合は行わず、代表パターンでテストを書く。

## 4. 料率マスタ設計（差し替え可能性の中核）

年度マスタを単一ソースに集約する。`src/lib/constants/rateMaster/2026.ts` が `RateMaster` 型を満たす。

```
RateMaster = {
  year: number
  socialInsurance: {
    healthRate: 0.0985          // 協会けんぽ東京 2026 健康保険料率
    childCareRate: 0.0023       // 子ども・子育て支援金率（健保と合算して折半対象）
    careRate: 0.0162            // 介護保険料率（40〜65歳）
    pensionRate: 0.183          // 厚生年金
    bonusPensionCapPerMonth: 1_500_000   // 厚年 標準賞与額 月上限
    bonusHealthCapAnnual: 5_730_000      // 健保・介護 標準賞与額 年度累計上限
    grades: StandardRemunerationGrade[]  // 等級区分（min/max/standardMonthly）のみ固定値
    bonusMinForMonthlyTreatment: 4       // 賞与年4回以上は報酬扱い
  }
  incomeTax: {
    brackets: { upTo: number|null; rate: number; deduction: number }[]  // 超過累進
    salaryDeduction: ...        // 給与所得控除（令和8年分マスタ値）
    basicDeduction: ...         // 基礎控除（課税所得帯別、令和8年分マスタ値）
    reconstructionRate: 0.021   // 復興特別所得税
  }
  residentTax: {
    rate: 0.10
    perCapita: number           // 均等割
    basicDeduction: number      // 住民税基礎控除
    salaryDeduction: ...        // 給与所得控除（所得税と同様）
  }
}
```

- 健保の本人負担は `(healthRate + childCareRate) = 10.08%` を合算した率で折半。
- 折半端数は協会けんぽ方式（被保険者負担 50 銭以下切捨・50 銭超切上）。
- `getRateMaster(year)` で年度解決。MVP は 2026 のみ登録。
- 令和8年(2026)分の基礎控除・給与所得控除の正確な改正値はマスタへ出典コメント付きで投入し、正式値確定後はマスタ差し替えで対応。

## 5. 計算ロジック

### 5.1 標準報酬月額
月額報酬から該当等級を線形探索（`>= min && (max === null || < max)`）。

### 5.2 月給の社会保険料
本人負担 = 健保(＋子育て) + 介護(40〜65歳のみ) + 厚年、いずれも標準報酬月額 × 率 ÷ 2。会社負担は原則同額。年間 = 月額 × 12。

### 5.3 賞与の社会保険料
- 標準賞与額 = 1,000 円未満切り捨て。
- 厚年: `min(標準賞与額, 1,500,000)`（月上限）。
- 健保・介護: 年度累計 5,730,000 円上限として残枠を消費（複数賞与を順次処理）。
- 賞与年 4 回以上は社会保険上は報酬扱い（`月給 + 年間賞与/12` を標準報酬月額の基礎にする）。

### 5.4 所得税
`給与収入 → 給与所得控除 → 給与所得`、`所得控除 = 基礎 + 社保 + iDeCo個人 + 小規模 + 配偶者 + 扶養`、`課税所得 = max(0, 給与所得 - 所得控除)`、`所得税 = 超過累進`、`合計 = 所得税 × 1.021`。

### 5.5 住民税
`課税標準 = 給与所得 - 住民税基礎控除 - 社保 - iDeCo個人 - 小規模 - その他`、`住民税 = 課税標準 × 10% + 均等割`。

### 5.6 限界税率
iDeCo+/小規模の節税効果に使う限界所得税率は、課税所得から該当ブラケットの率を引く。

## 6. iDeCo+

会社負担分と個人負担分を必ず分離する。

- 会社負担分: 法人で全額損金 / 個人の給与課税なし / 社保対象外 / 所得税・住民税対象外。
- 個人負担分: 小規模企業共済等掛金控除（所得税・住民税が下がる）/ 給与から拠出のため**社保削減効果は 0**。
- バリデーション: 合計（会社＋個人）月額 5,000〜23,000 円、1,000 円単位。

## 7. 小規模企業共済
個人の所得控除（所得税・住民税減）。社保削減効果なし。月額 0〜70,000 円。

## 8. 社宅（概算）
`会社負担家賃年額 = 月額 × 会社負担率 × 12`、経済的メリット = 会社負担家賃年額。注記「概算。役員社宅の賃貸料相当額は固定資産税評価額等で変わる」を表示。

## 9. 出張旅費・日当（概算）
`年額 = 日数/月 × 日当 × 12`。法人=損金 / 個人=非課税で実質手取りに加算 / 社保対象外。注記「旅費規程・出張実態・金額妥当性が必要」。

## 10. 法人税
`法人利益 = preSalaryProfit - 役員報酬 - 事前確定届出給与 - 会社社保 - iDeCo+会社分 - 会社負担家賃 - 旅費 - 経営セーフティ共済 - その他損金`、`法人税等 = max(0, 利益) × 実効税率(入力, 既定30%)`、`法人残キャッシュ = 利益 - 法人税`。

## 11. 実質手取り

- 会社員: `現金手取り = 年収 - 社保 - 所得税 - 住民税 - iDeCo`、`実質 = 現金 + 家賃補助 + 企業型DC効果`。
- 法人役員: `現金手取り = 役員報酬+賞与 - 社保 - 所得税 - 住民税 - iDeCo個人 - 小規模`、`実質 = 現金 + 社宅 + 旅費 + iDeCo+会社分 + 小規模`。
- 表示は **現金手取り / 実質手取り / 将来資産込み実質手取り** の 3 区分。

## 12. 比較結果テーブル
項目（給与収入・社保本人/会社・所得税・住民税・法人税・社宅・iDeCo/iDeCo+・小規模・出張旅費・現金手取り・実質手取り・法人残・個人＋法人合計）× 会社員 / 法人役員 / 差額。

## 13. 最適化モード
役員報酬（月給）と賞与を総当たり。スコア = `directorEffectiveNet + corporateRemainingCash`。降順ソート上位 20 件を返す。総当たり件数が過大にならないようステップを検証。
